# Project Onboarding Questions & Analysis

Use this guide to verify the tech stack and fill in any gaps after the initial analysis.

## 🕵️ Analysis Verification (Agent-led)
- **Detected Stack**: Is the stack I identified (e.g., Laravel, React) correct?
- **Proposed Stack**: If I suggested a stack for a new project, does it meet your requirements?
- **Main Path**: Is the identified module path (e.g., `/backend`) the primary focus?

## 📁 General Information (Missing Data)
- **Project Name**: What is the official name of this project?
- **Description**: What is the primary purpose of this project?
- **Repository URL**: Where is the source code hosted?

## 🛠 Deep Dive (If needed)
- **Database**: Which database system is in use?
- **Additional Agents**: Are there any specific roles needed (e.g., DevOps, FrontendExpert)?
- **Specific Standards**: Are there any project-specific coding or git standards?

## 🚀 Deployment & Environments
- **Environments**: What environments exist (e.g., staging, production)?
- **Deployment Strategy**: How is the project deployed?
